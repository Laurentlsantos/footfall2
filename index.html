Preface

This tutorial book is intended to provide a hands-on introduction to human mobility analytics with fully anomimised data collected through mobile phones and provided by the geo-location database service Visitor Insights1. Based on real life data, a full range of analysis and data visualisation are provided, uncovering insights from social movements.The book aims to provide a handful framework for analysing human movements and interpreting the results with high quality charts and maps.

With the help of sensors and online networks, the study of human mobility can be applied on many different situations and provide detailed information about people (or visitors2) and locations. The development of communication technologies and the diffusion of mobile devices gives an unprecedented opportunity to study massive data of human behaviour by means of databases created to store location data and their associated attributes.

Unraveling human movement helps answering key questions including what locations drive visitors, who is the visitor of a commercial or urban location and what is the visitor behaviour in his journey.

The book has been structured as a resource tool for the statistical computing and graphics language called R. R provides a wide variety of statistical and graphical techniques, and is highly extensisive. The concepts and tools are taught on the same frame, so you can follow the examples and apply the concepts at the same time. R is free and it is a broad-based application for statistical analysis in general.

Conventions

Throughout the book we use a consistent set of conventions to refer to code:

  • Functions are in a code font and followed by parentheses, like sum(), or mean().

  • Used for program listings to refer to program elements such as variable or function names, databases, data types, element variables, statements and keyword, are in a code font, without parentheses, like data or x.

  • If we want to make it clear what package an object comes from, we’ll use the package name followed by two colons, like dplyr::mutate().

We also employ some specific typographical conventions:

  • italic indicates new terms, reference sources, file names and file extensions.

  • bold indicates the name of R packages.

  • Constant width italic shows text that should be replaced with use-supplied values or by values determined by context.

For your convenience (e.g. when you want to copy and run the code), we do not add prompts (> and +) to R source code in this book and outputs are preceded by ##. These are the results of running the code and should not be typed in the console or included in your scripts. You should see these outputs in your R Studio console when you run the code.
